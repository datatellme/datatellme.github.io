    batch script로 자동화 프로그램 만들기

    들어가며

    • batch 파일(배치 파일)은 Windows 운영 체제에서 명령어를 자동으로 실행하는 스크립트 파일입니다.

    • batch 파일을 만드려면 기본적인 Windows 명령어와 스크립트 작성 방법을 이해해야 합니다.

    배치파일 기본

    Windows 명령 프롬프트 (Command Prompt) 기본 명령어:

    • echo: 메시지를 출력, cd: 디렉토리 변경, dir: 디렉토리 내용 보기, copy: 파일 복사, del: 파일 삭제, pause: 일시 정지, set: 변수 설정

    배치 파일의 구조

    • batch 파일은 명령 프롬프트에서 실행할 수 있는 명령어들을 줄 단위로 나열합니다.
    @echo off
echo Hello, World!
pause

    제어 흐름과 논리

    • 조건문: if 명령어를 사용하여 조건에 따라 명령을 실행.
    if exist "test.txt" echo File exists!
    • 반복문: for 명령어로 반복 작업 수행.
    for %%f in (*.txt) do echo %%f

    환경 변수

    • 시스템 및 사용자 정의 환경 변수 활용.
    set MY_VAR=Hello
echo %MY_VAR%

    외부 프로그램 호출

    • .exe 파일, Python 스크립트, R 스크립트 등 다른 프로그램을 배치 파일에서 호출하는 방법.
    python script.py

    입출력 리다이렉션

    • 명령 실행 결과를 파일로 저장하거나 다른 입력으로 넘기는 방법.
    dir > output.txt

    디버깅과 테스트

    • 배치 파일을 작성한 후 명령 프롬프트에서 실행하며 디버깅하는 방법.

    • @echo on을 통해 명령 실행 과정을 확인.

    MkDocs 홈페이지 제작 실무 적용

    • MkDocs 홈페이지 제작시 아래의 절차로 자동화 batch 파일을 만들어 원클릭으로 홈페이지의 변경된 내용을 업로드할 수 있습니다.

    홈패이지 블로그 틀 파이썬으로 실행

    @echo off
"C:\Users\oem\miniconda3\envs\myfirstenv\python.exe" C:\gppj\auto\mac_01_gppj01_040304010202.py
pause

    R에서 Rmd를 Md로 전환하는 Knit 실행

    @echo off
"C:\R\R-4.4.0\bin\x64\Rscript.exe" "C:\gppj\auto\mac_02_knit_rmd04.R"
pause

    MkDocs 빌드

    @echo off
"C:\Users\oem\miniconda3\envs\myfirstenv\python.exe" -m mkdocs build -f C:\gppj\data-tell-me_ver03\mkdocs.yml
pause
    MkDocs 서버 실행

    @echo off
"C:\Users\oem\miniconda3\envs\myfirstenv\python.exe" -m mkdocs serve -f C:\gppj\data-tell-me_ver03\mkdocs.yml
pause

    A폴더의 내용을 B 폴더에 모두 복사

    • 아래는 C:\gppj\data-tell-me_ver04\site에 있는 모든 파일을 C:\gppj\data-tell-me_ver05로 복사하는 Batch 스크립트입니다.
    @echo off
:: 복사할 원본 디렉터리 경로
set source="C:\gppj\data-tell-me_ver04\site"

:: 복사 대상 디렉터리 경로
set destination="C:\gppj\data-tell-me_ver05"

:: 대상 디렉터리가 없으면 생성
if not exist %destination% (
    echo Target directory does not exist. Creating: %destination%
    mkdir %destination%
)

:: 모든 파일 복사
xcopy /e /i /y %source% %destination%

:: 완료 메시지
echo All files have been successfully copied to %destination%.
pause

    • 스크립트 설명:

    • set source와 set destination: 원본 디렉터리와 대상 디렉터리의 경로를 설정합니다.

    • if not exist: 대상 디렉터리가 없는 경우, mkdir 명령어로 디렉터리를 생성합니다.

    • xcopy 명령어:

    • /e: 하위 디렉터리를 포함한 모든 파일 복사.

    • /i: 복사 대상이 디렉터리임을 지정.

    • /y: 파일 덮어쓰기 확인을 생략.

    • pause: 작업 완료 후 창이 바로 닫히지 않도록 합니다.

    • 사용 방법: 위 코드를 복사해 .bat 확장자로 저장하세요. 예: copy_files.bat.

    • 파일을 실행하면 복사가 진행됩니다.

    Sourcetree애서 커밋과 푸시하기

    • 이 스크립트는 Sourcetree를 실행하고, Git을 통해 현재 디렉토리의 변경 사항을 날짜와 시간으로 커밋한 다음, 푸시합니다.
    @echo off

REM Sourcetree 실행
start "" "C:\Program Files\Sourcetree\Sourcetree.exe"

REM Git 작업 디렉토리 설정 (현재 디렉토리 기준)
cd /d C:\path\to\your\repository

REM 현재 날짜와 시간 가져오기
for /f "tokens=2 delims==" %%i in ('"wmic os get localdatetime /value"') do set datetime=%%i
set datetime=%datetime:~0,4%-%datetime:~4,2%-%datetime:~6,2% %datetime:~8,2%:%datetime:~10,2%:%datetime:~12,2%

REM Git 상태 확인
git status

REM 모든 변경사항 스테이징
git add .

REM 커밋 메시지 작성 (날짜와 시간 포함)
git commit -m "Automated commit on %datetime%"

REM 원격 저장소로 푸시
git push

REM 완료 메시지
echo 작업이 완료되었습니다.
pause

    • 사용법

    • 경로 수정: C:\Program Files\Sourcetree\Sourcetree.exe 경로를 Sourcetree 설치 위치로 수정합니다. C:\path\to\your\repository를 Git 저장소의 경로로 수정합니다.

    • 저장: 위 코드를 .bat 파일로 저장합니다. 예: git_push.bat.

    • 실행: .bat 파일을 실행하면 Sourcetree가 열리고, Git 명령어가 자동으로 실행되어 커밋 및 푸시가 이루어집니다.

    • 주요 기능: Sourcetree 실행. 현재 디렉토리의 모든 변경사항을 날짜와 시간으로 커밋. 푸시를 통해 원격 저장소 업데이트.

    결어

    • batch script로 심플한 실행 파일을 만들 수 있습니다.

    • 윈도우 작업스케줄러와 함께 사용하면 시간에 따라 자동으로 실행 할 수 있게 세팅 할 수 있습니다.
