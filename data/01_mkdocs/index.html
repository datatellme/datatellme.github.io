    MkDocs으로 나의 홈페이지 만들기(with RStudio)

    왜 MkDocs인가?

    • 홈페이지를 무료로 만들 수 있습니다.
    • 마크다운을 이용해서 쉽게 만들 수 있습니다.
    • 홈페이지에 북다운으로 작성한 책을 출판할 수 있습니다.
    • 홈페이지가 매우 보기 좋고 각각의 기기에서 조회 가능합니다.

    홈페이지 작업환경 만들기

    R 프로그램 설치

    • 설치전 준비
    • 컴퓨터의 이름은 한글로 하지 않고 영어로 지정합니다.
    • 사용자 이름도 한글로 하지 않고 영어로 지정합니다.

    • 폴더 이름도 한글로 하지 않고 영어로 지정합니다.

      • 특수문자나 공백을 사용하지 않습니다.

    • 설치 절차

    • www.r-project.org에 접속합니다.
    • R 홈페이지의 화면 왼쪽의 Download의 CRAN을 선택합니다.
    • 아래와 같이 CRAN Mirrors가 나타밥니다.
      • 총 47개국의 122개 서버목록 중 한국 웹주소 선택합니다.
    • 컴퓨터 운영체제 선택합니다.
    • 현재의 최신 버전 다운로드 합니다.
    • 다운로드 완료 후 관리자 모드로 프로그램 설치합니다.
    • 프로그램 설치시 기본 설정 값으로 다음 클릭합니다.
    • R 프로그램의 설치 위치 C:\R을 새롭게 만들어 설치합니다.
    • 모두 체크하고 다음 버튼 클릭합니다.

    RStudio 프로그램 설치

    • 설치 절차
    • www.r-project.org에 접속합니다.
    • R 홈페이지의 화면 왼쪽의 Download의 CRAN을 선택합니다.
    • 아래와 같이 CRAN Mirrors가 나타남니다.
      • 총 47개국의 122개 서버목록 중 한국 웹주소 선택합니다.
    • 컴퓨터 운영체제 선택합니다.
    • 현재의 최신 버전 다운로드합니다.
    • 다운로드 완료 후 관리자 모드로 프로그램 설치합니다.
    • 프로그램 설치시 기본 설정 값으로 다음 클릭합니다.
    • R 프로그램의 설치 위치 C:\R을 새롭게 만들어 설치합니다.
    • 모두 체크하고 다음 버튼 클릭합니다.

    powershell 프로그램 설치

    python 설치시 miniconda 설치 이유

    • 프로젝트 별로 가상 환경을 만들 수 있습니다.
    • python의 다른 버전을 각각의 프로젝트별로 설치 가능합니다.
    • 프로젝트별 패키지 버전관리가 가능합니다.

    miniconda 프로그램 설치

    • Miniconda 홈페이지에서 다운로드합니다.
    • 다운로드 후 인스톨 실행 [Next]클릭 다음하면 [I Agree] 클릭합니다.
    • 인스톨타임 All Users 산택 후 [Next] 클릭합니다.
    • 인스톨 경로 디폴트값으로 선택하고 [Next] 클릭합니다.
    • 인스톨 옵션창에서 Path 설정 체크박스 모두 체크 후 [Install] 클릭합니다.
    • 설치 완료되면 Win+R 를 칩니다.
    • 실행창에 cmd 입력한 후 Enter 를 칩니다.
    • cmd창 열리면 'python' 입력한 후 Enter 후 파이선 작동하는지 확인합니다.

    miniconda 작업 순서(CMD에서 작업)

    • Win+R 실행창에 cmd 입력한 후 Enter

    • miniconda 실행해서 가상공간(홈페이지 작업 가상공간 ) 생성합니다.

    C:\users\psnam\conda create -n myhomepienv python=3.10
    • 만들어진 가상공간으로 들어갑니다.
    C:\users\psnam\conda activate myhomepienv
    • 들어간 가상공간에서 설치된 프로그램패키지 확인합니다.
    (myhomepienv) C:\users\psnam\conda list
    • 들어간 가상공간에서 mkdocs 프로그램패키지 설치합니다.
    (myhomepienv) C:\users\psnam\pip install mkdocs
    • 만들어진 가상공간으로 나업니다.
    C:\users\psnam\conda deactivate
    • 만들어진 가상공간 삭제합니다.
    C:\users\psnam\conda env remove -n myhomepienv
    • 가상공간 안의 리스트 확인합니다.
    C:\users\psnam\conda env list

    MkDocs 설치(CMD에서 작업)

    • Win+R 실행창에 cmd 입력한 후 Enter 를 칩니다.
    C:\users\psnam\
    • conda 실행해서 홈페이지 작업 가상 공간 입장합니다.
    C:\users\psnam\conda activate myhomepienv
    • mkdocs는 MkDocs 설치합니다.
    • mkdocs-material은 material 테마 설치합니다.
    • pymdown-extensions는 PyMdown을 설치합니다.
     (myhomepienv) C:\users\psnam\pip install mkdocs   
 (myhomepienv) C:\users\psnam\pip install mkdocs-material
 (myhomepienv) C:\users\psnam\pip install pymdown-extensions
    • mkdocs 새프로젝트(홈피 이름) 만듭니다.
    (myhomepienv) C:\users\psnam\mkdocs new myfirsthomepi   # myfirsthomepi라는 프로젝트 생성

    MkDocs 작업 절차(RStudio에 작업)

    • RStudio 프로그램을 실행합니다.
    • RStudio project를 새로 mkdocs 새프로젝트(홈피 이름)로 생성된 c:\users\psnam\myfirsthomepi 폴더와 연결합니다.

    • RStudio의 메뉴바 Tools > Terminal > Terminal Options > General 에서 New Terminal open에 Custoum 크릭하고 Powershell 연결하고 적용, 확인하고 밖으로 나옵니다.

    • 터미널에 powershell이 연결

    (basic) C:\users\psnam\myfirsthomepi
    • RStudio 터미널(powershell)에서 홈페이지 작업 가상 공간 입장합니다.
    (basic) C:\users\psnam\myfirsthomepi\conda activate myhomepienv
    • 로컬 서버 띄우기 -> http://localhost:8000/ 나 http://127.0.0.1:8000/ 브라우저에서 확인 가능합니다.
     (myhomepienv) C:\users\psnam\myfirsthomepi\mkdocs serve
    • html 문서 만듭니다.
     (myhomepienv) C:\users\psnam\myfirsthomepi\mkdocs build

    프로젝트파일 구조

    • mkdocs new 으로 새로운 프로젝트를 생성하는데 그 구조는 다음과 같습니다.
    <myfirsthomepi> 
    + docs
         -  index.md
    - mkdocs.yml

    생성된 파일 수정

    • RStudio 프로젝트 연결 후 마크다운 파일은 docs 폴더 아래 위치하고 mkdocs.yml을 통해 설정을 수행합니다.

    • 다음은 생성된 docs의 추리입니다.

    docs 
   - index.md
   - about.md
   + chapter1 
         - 1-1.md
         - 1-2.md

    mkdocs.yml 파일의 내용

    nav: 는 홈페이지 목차

    site_name: My Docs
site_url: https://example.com
nav:
  - Home: index.md
  - About: about.md
  - chapter1:
      - 1-1 : chaper1/1-1.md
      - 1-2 : chaper1/1-2.md    
theme : 
    name: 'material'
    • pages 아래에 계층구조로 지정하면 됩니다.
    • pages가 없어도 docs 디렉토리의 계층구조에 따라 자동으로 만들어 주나, 문서의 #(없으면 파일명)이 제목으로 설정됩니다.
    • 따라서 pages를 사용하여 별도의 목차를 구성하는 것입니다.
    • 파일 명에 공백 등이 있는 경우 따옴표로 묶어 줍니다 - 1-1 : 'chaper1/1-1.md'

    • theme은 테마를 지정하는데 지정하지 않을 경우 mkdocs 테마가 지정됩니다. mkdocs, readthedocs 테마를 제외한 테마는 MkDocs Wiki 에서 구할 수 있습니다.

    • material : 저는 material 테마를 추천합니다.

    theme 내용 추가

    • mkdocs-material를 설치한 후 theme: 아래와 같이 작성하면 지금 나의 홈페이지와 같은 환경이 구축됩니다.
    theme: 
   name: 'material'
   features:
    - navigation.tabs
    - navigation.sections
    - toc.integrate
    - navigation.top
   language: en
   palette:
    - scheme: default
      toggle:
        icon: material/toggle-switch-off-outline 
        name: Switch to dark mode
      primary: blue
      accent: purple 
    - scheme: slate 
      toggle:
        icon: material/toggle-switch
        name: Switch to light mode    
      primary: teal
      accent: lime    
   logo: img/peterpark_01.jpg

    markdown_extensions 추가

    • markdown_extensions 을 설치한 후 mkdocs.yml에 추가합니다.
    markdown_extensions:
  - pymdownx.superfences
  - attr_list
  - admonition
  - pymdownx.details
  - pymdownx.keys
  - pymdownx.critic
  - pymdownx.emoji:
      emoji_generator: !!python/name:pymdownx.emoji.to_svg
  - pymdownx.arithmatex
  - pymdownx.betterem:
      smart_enable: all
  - pymdownx.caret
  - pymdownx.inlinehilite
  - pymdownx.magiclink
  - pymdownx.mark
  - pymdownx.smartsymbols
  - pymdownx.tasklist:
      custom_checkbox: true
  - pymdownx.tilde

    수식 설정

    extra_javascript:
  - 'https://cdnjs.cloudflare.com/ajax/libs/mathjax/2.7.5/MathJax.js?config=TeX-MML-AM_CHTML'
    • 이제 마크다운 문서에서 ...이나 ... 으로 Latex 수식을 적용할 수 있습니다.
    • MathJax의 CDN 서버는 버전에 따라 조금씩 바뀝니다(위의 2.7.5는 2020.1.14기준이며, mathJax 홈페이지에서 확인할 수 있다)
    • Markdown 수식에서 \rm 대신 \textrm을 쓰도록 합니다(Markdown문서를 Pandoc으로 변환할 때 경우 따라 발생하는 오류 유발 방지합니다.)
