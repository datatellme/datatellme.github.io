    MkDocs으로 나의 홈페이지 만들기(with RStudio)

    들어가며

    왜 MkDocs인가?

    • 홈페이지를 무료로 만들 수 있습니다.
    • 마크다운을 이용해서 쉽게 만들 수 있습니다.
    • 홈페이지에 북다운으로 작성한 책을 출판할 수 있습니다.
    • 홈페이지가 매우 보기 좋고 각각의 기기에서 조회 가능합니다.

    MkDocs 홈페이지 만드는 방법

    사전작업: miniconda 작업 순서(CMD에서 작업)

    • Win+R 실행창에 cmd 입력한 후 Enter

    • miniconda 실행해서 가상공간(홈페이지 작업 가상공간 ) 생성합니다.

    C:\users\psnam\conda create -n myhomepienv python=3.10
    • 만들어진 가상공간으로 들어갑니다.
    C:\users\psnam\conda activate myhomepienv
    • 들어간 가상공간에서 설치된 프로그램패키지 확인합니다.
    (myhomepienv) C:\users\psnam\conda list
    • 들어간 가상공간에서 mkdocs 프로그램패키지 설치합니다.
    (myhomepienv) C:\users\psnam\pip install mkdocs
    • 만들어진 가상공간으로 나업니다.
    C:\users\psnam\conda deactivate
    • 만들어진 가상공간 삭제합니다.
    C:\users\psnam\conda env remove -n myhomepienv
    • 가상공간 안의 리스트 확인합니다.
    C:\users\psnam\conda env list

    MkDocs 설치(CMD에서 작업)

    • Win+R 실행창에 cmd 입력한 후 Enter 를 칩니다.
    C:\users\psnam\
    • conda 실행해서 홈페이지 작업 가상 공간 입장합니다.
    C:\users\psnam\conda activate myhomepienv
    • mkdocs는 MkDocs 설치합니다.
    • mkdocs-material은 material 테마 설치합니다.
    • pymdown-extensions는 PyMdown을 설치합니다.
     (myhomepienv) C:\users\psnam\pip install mkdocs   
 (myhomepienv) C:\users\psnam\pip install mkdocs-material
 (myhomepienv) C:\users\psnam\pip install pymdown-extensions
    • mkdocs 새프로젝트(홈피 이름) 만듭니다.
    (myhomepienv) C:\users\psnam\mkdocs new myfirsthomepi   # myfirsthomepi라는 프로젝트 생성

    MkDocs 작업 절차(RStudio에 작업)

    • RStudio 프로그램을 실행합니다.
    • RStudio project를 새로 mkdocs 새프로젝트(홈피 이름)로 생성된 c:\users\psnam\myfirsthomepi 폴더와 연결합니다.

    • RStudio의 메뉴바 Tools > Terminal > Terminal Options > General 에서 New Terminal open에 Custoum 크릭하고 Powershell 연결하고 적용, 확인하고 밖으로 나옵니다.

    • 터미널에 powershell이 연결

    (basic) C:\users\psnam\myfirsthomepi
    • RStudio 터미널(powershell)에서 홈페이지 작업 가상 공간 입장합니다.
    (basic) C:\users\psnam\myfirsthomepi\conda activate myhomepienv
    • 로컬 서버 띄우기 -> http://localhost:8000/ 나 http://127.0.0.1:8000/ 브라우저에서 확인 가능합니다.
     (myhomepienv) C:\users\psnam\myfirsthomepi\mkdocs serve
    • html 문서 만듭니다.
     (myhomepienv) C:\users\psnam\myfirsthomepi\mkdocs build

    MkDocs 프로젝트

    프로젝트파일 구조

    • mkdocs new 으로 새로운 프로젝트를 생성하는데 그 구조는 다음과 같습니다.
    <myfirsthomepi> 
    + docs
         -  index.md
    - mkdocs.yml

    생성된 파일 수정

    • RStudio 프로젝트 연결 후 마크다운 파일은 docs 폴더 아래 위치하고 mkdocs.yml을 통해 설정을 수행합니다.

    • 다음은 생성된 docs의 추리입니다.

    docs 
   - index.md
   - about.md
   + chapter1 
         - 1-1.md
         - 1-2.md

    mkdocs.yml 파일의 내용

    nav: 는 홈페이지 목차

    site_name: My Docs
site_url: https://example.com
nav:
  - Home: index.md
  - About: about.md
  - chapter1:
      - 1-1 : chaper1/1-1.md
      - 1-2 : chaper1/1-2.md    
theme : 
    name: 'material'
    • pages 아래에 계층구조로 지정하면 됩니다.
    • pages가 없어도 docs 디렉토리의 계층구조에 따라 자동으로 만들어 주나, 문서의 #(없으면 파일명)이 제목으로 설정됩니다.
    • 따라서 pages를 사용하여 별도의 목차를 구성하는 것입니다.
    • 파일 명에 공백 등이 있는 경우 따옴표로 묶어 줍니다 - 1-1 : 'chaper1/1-1.md'

    • theme은 테마를 지정하는데 지정하지 않을 경우 mkdocs 테마가 지정됩니다. mkdocs, readthedocs 테마를 제외한 테마는 MkDocs Wiki 에서 구할 수 있습니다.

    • material : 저는 material 테마를 추천합니다.

    theme 내용 추가

    • mkdocs-material를 설치한 후 theme: 아래와 같이 작성하면 지금 나의 홈페이지와 같은 환경이 구축됩니다.
    theme: 
   name: 'material'
   features:
    - navigation.tabs
    - navigation.sections
    - toc.integrate
    - navigation.top
   language: en
   palette:
    - scheme: default
      toggle:
        icon: material/toggle-switch-off-outline 
        name: Switch to dark mode
      primary: blue
      accent: purple 
    - scheme: slate 
      toggle:
        icon: material/toggle-switch
        name: Switch to light mode    
      primary: teal
      accent: lime    
   logo: img/peterpark_01.jpg

    markdown_extensions 추가

    • markdown_extensions 을 설치한 후 mkdocs.yml에 추가합니다.
    markdown_extensions:
  - pymdownx.superfences
  - attr_list
  - admonition
  - pymdownx.details
  - pymdownx.keys
  - pymdownx.critic
  - pymdownx.emoji:
      emoji_generator: !!python/name:pymdownx.emoji.to_svg
  - pymdownx.arithmatex
  - pymdownx.betterem:
      smart_enable: all
  - pymdownx.caret
  - pymdownx.inlinehilite
  - pymdownx.magiclink
  - pymdownx.mark
  - pymdownx.smartsymbols
  - pymdownx.tasklist:
      custom_checkbox: true
  - pymdownx.tilde

    수식 설정

    extra_javascript:
  - 'https://cdnjs.cloudflare.com/ajax/libs/mathjax/2.7.5/MathJax.js?config=TeX-MML-AM_CHTML'
    • 이제 마크다운 문서에서 ...이나 ... 으로 Latex 수식을 적용할 수 있습니다.
    • MathJax의 CDN 서버는 버전에 따라 조금씩 바뀝니다(위의 2.7.5는 2020.1.14기준이며, mathJax 홈페이지에서 확인할 수 있다)
    • Markdown 수식에서 \rm 대신 \textrm을 쓰도록 합니다(Markdown문서를 Pandoc으로 변환할 때 경우 따라 발생하는 오류 유발 방지합니다.)

    홈페이지 내용 업데이트

    업데이트 절차

    • 깃허브에 홈페이지를 올리기 위해서는 깃허브의 본인 계정이 있어야 합니다.
    • 깃허브 본인 계정과 RStutio를 연동 시키는 RStudio의 프로젝트를 만듭니다.(ex: C:>RPJ>study>datatellme.github.io)
    • Mkdoc(ex: C:>사용자>psnam> data-tell-me) 폴더 안의 docs폴더의 md파일을 수정합니다.
    • 수정이 완료되면 Terminal(powershell)에서 html 문서를 만듭니다.
     (myhomepienv) C:\users\psnam\myfirsthomepi\mkdocs build

    • 수정되어 변경되고 만들어진 html 파일은 site 폴더에 있습니다.

    • RStudio의 프로젝트 폴더안에 위에서 수정하고 만들어지 html 파일을 생성한 Mkdoc(ex: C:>사용자>psnam> data-tell-me > site) 홈페이지 파일들을 복사합니다.

    • 정리하면 1차적으로 처음 Mkdoc로 만든 홈페이지(data-tell-me>docs) 폴더의 md파일을 수정하고 그 파일을 html 파일로 변환하고 변환된 html 파일이 있는 (data-tell-me>site)폴더에서 복사하여 datatellme.github.io폴더에 업데이트합니다.

    • 최종적으로 게시는 datatellme.github.io의 RStudio 프로젝트 파일을 실행하면 변경된 내용이 체크되어 있고 push하면 내용 수정이 완료됩니다.

    • 이때 RStudio 프로젝트에서 push할 때 반드시 commit에 내용을 적어 commit해주고 push하여야 합니다.

    깃허브와 RStutio 연동 방법

    • Git을 설치합니다.(싸이트: https://git-scm.com/)

    • 설정은 모두 기본 값으로 하시면 됩니다.

    • Terimnal 창에서 git을 실행하여 정상적으로 설치되었는지 유무를 확인합니다.

    • Github에 회원가입을 합니다

    • Github 저장소를 만듭니다.

    • RStudio Tools>Global Option>Git/SVN으로 들어가 git이 저장된 파일 디렉토리가 맞는지 확인하고, create SSH Key를 클릭하여 키를 생성합니다.

    • RStudio의 View public key를 클릭하면 ssh-rsa로 시작하는 public key가 새창에 뜨고 그것을 복사합니다.

    • Github settings: 깃허브로 와서 오른쪽 상단 계정이름/아이디를 클릭합니다.

    • Github 세팅으로 들어가 SSH and GPG Keys에 들어가 SSH keys 생성을 클릭하여 key 이름을 정해주고 복사한 RStudio ssh-rsa로 시작하는 public key를 붙여넣기 합니다.

    • RStudio에서 새프로젝트를 클릭한 후 아래 화면에서 Version Control을 클릭합니다.

    • github에서 주소 SSH 복사를 클릭합니다.

    • 복사한 github 주소를 RStudio에서 새프로젝트 빈란에 붙여 넣습니다.

    깃허브와 RStutio 연동시 에러 핸들링

    • 첫번째 이슈: 로그인

    아래 메시지를 확인합니다.

     *** Please tell me who you are.
Run
  git config --global user.email "you@example.com"
  git config --global user.name "Your Name"
to set your account's default identity.
Omit --global to set the identity only in this repository.

    위 메시지는 git commit을 할 때 발생합니다.

    위 메시지는 쉽게 말해 로그인을 하라는 뜻입니다.

    해결 방법: 두개의 git config을 각각 복사하여 파워셀어서 실행하며 이 때 본인의 깃허브 메일과 아이디를 입력하시면 됩니다.

    • 두번째 이슈: 깃허브 싸이트에서 수정

    웹사이트에서 무언가를 수정하고 이후에 git push를 시도합니다. 이 때 아래와 같은 메시지가 출력됩니다.

    git push 

To https://github.com/yourname/foo.git

 ! [rejected]        master -> master (non-fast-forward)
error: failed to push some refs to 'https://github.com/yourname/foo.git'
hint: Updates were rejected because the tip of your current branch is behind
hint: its remote counterpart. Integrate the remote changes (e.g. hint: 'git pull ...') before pushing again.
hint: See the 'Note about fast-forwards' in 'git push --help' for details.

    해결 방법: git pull을 클릭합니다. 그리고 처음부터 파일 수정, commit, push를 진행하면 됩니다.

    홈페이지 내용 업데이트 자동화(Sourcetree)

    자동화 환경 세팅

    • 깃허브 업로드 프로그램인 Sourcetree를 설치한다.

    • 홈페이지 내용 수정파일 폴더와 업로드 파일 폴더를 구분

    • 내용 수정 폴더: C:\gppj\data-tell-me_ver04

    • 업로드 폴더: C:\gppj\data-tell-me_ver05

    • 내용 수정 폴더의 docs폴더에 마크다운 파일을 수정한다.

    • 자동화를 위한 배치 파일을 C:\gppj\auto_ver04 폴더에 작성한다.

    • 배치 파일명은 다음과 같이 예를 들어 명명한다.

    • 04_homp_mkdocs_build.bat

    • 05_homp_mkdocs_serve.bat

    • 06_copy_MkDocfiles.bat

    • 07_git_push.bat

    빌드 배치 파일 작성

    • 04_homp_mkdocs_build.bat
    @echo off
"C:\Users\oem\miniconda3\envs\myfirstenv\python.exe" -m mkdocs build -f C:\gppj\data-tell-me_ver04\mkdocs.yml
pause

    서버 배치 파일 작성

    • 05_homp_mkdocs_serve.bat

"C:\Users\oem\miniconda3\envs\myfirstenv\python.exe" -m mkdocs serve -f C:\gppj\data-tell-me_ver04\mkdocs.yml

    복사 배치 파일 작성

    • 06_copy_MkDocfiles.bat
    @echo off
:: 복사할 원본 디렉터리 경로
set source="C:\gppj\data-tell-me_ver04\site"

:: 복사 대상 디렉터리 경로
set destination="C:\gppj\data-tell-me_ver05"

:: 대상 디렉터리가 없으면 생성
if not exist %destination% (
    echo Target directory does not exist. Creating: %destination%
    mkdir %destination%
)

:: 모든 파일 복사
xcopy /e /i /y %source% %destination%

:: 완료 메시지
echo All files have been successfully copied to %destination%.
pause

    Sourcetree git push 배치 파일 작성

    • 07_git_push.bat
    @echo off

REM Sourcetree 실행
start "" "C:\Users\oem\AppData\Local\SourceTree\app-3.4.21\Sourcetree.exe"

REM Git 작업 디렉토리 설정 (현재 디렉토리 기준)
cd /d C:\gppj\data-tell-me_ver05

REM 현재 날짜와 시간 가져오기
for /f "tokens=2 delims==" %%i in ('"wmic os get localdatetime /value"') do set datetime=%%i
set datetime=%datetime:~0,4%-%datetime:~4,2%-%datetime:~6,2% %datetime:~8,2%:%datetime:~10,2%:%datetime:~12,2%

REM Git 상태 확인
git status

REM 모든 변경사항 스테이징
git add .

REM 커밋 메시지 작성 (날짜와 시간 포함)
git commit -m "Automated commit on %datetime%"

REM 원격 저장소로 푸시
git push

REM 완료 메시지
echo 작업이 완료되었습니다.
pause
