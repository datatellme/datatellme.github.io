    자동화 작업 환경 설정

    들어가며

    파이썬 통계 분석에 R 및 RStudio를 사용하는 이유는 다음과 같습니다. - 분석 확장성 높습니다. 즉 R 분석 및 python 분석이 동시에 가능합니다. - Rmd 파일 작성으로 북다운, 홈페이지, 블로그. pdf등 다양한 문서작업이 가능 합니다. - tiny텍스 등 레이텍 사용이 가능해 양질의 pdf 책등을 만들 수 있습니다. - 재생가능한 연구를 수행할 수 있습니다.

    R 프로그램 설치

    • R 프로그램의 공식 웹사이트(https://www.r-project.org/)에서 운영 체제에 맞는 설치 파일을 다운로드합니다.
    • 다운로드가 완료되면 설치 파일을 실행합니다.
    • 설치 과정에서 제시되는 지시에 따라 설치를 진행합니다. 보통은 기본적인 설정으로 설치가 가능합니다.

    설치전 준비

    • 컴퓨터의 이름은 한글로 하지 않고 영어로 지정합니다.
    • 사용자 이름도 한글로 하지 않고 영어로 지정합니다.
    • 폴더 이름도 한글로 하지 않고 영어로 지정합니다.
    • 특수문자나 공백을 사용하지 않습니다.

    설치 절차

    • www.r-project.org에 접속합니다.
    • R 홈페이지의 화면 왼쪽의 Download의 CRAN을 선택합니다.
    • 아래와 같이 CRAN Mirrors가 나타 납니다.
    • 총 47개국의 122개 서버목록 중 한국 웹주소 선택합니다.
    • 컴퓨터 운영체제 선택합니다.
    • 현재의 최신 버전 다운로드 합니다.
    • 다운로드 완료 후 관리자 모드로 프로그램 설치합니다.
    • 프로그램 설치시 기본 설정 값으로 다음 클릭합니다.
    • R 프로그램의 설치 위치 C:\R을 새롭게 만들어 설치합니다.
    • 모두 체크하고 다음 버튼 클릭합니다.

    RTools 프로그램 설치

    • RTools의 공식 웹사이트(https://cran.r-project.org/bin/windows/Rtools/)에서 운영 체제에 맞는 설치 파일을 다운로드합니다.
    • 다운로드한 설치 파일을 실행합니다.

    • 설치 과정에서 제시되는 지시에 따라 설치를 진행합니다. 일반적으로 기본 설정을 유지하는 것이 좋습니다.

    • 설치가 완료되면 RTools를 사용하여 R 패키지를 빌드하고 설치할 수 있습니다.

    • RTools를 설치하면 R 패키지를 더 쉽게 관리하고 사용할 수 있습니다.

    설치 절차

    • 관리자 모드 실행하여 설치합니다
    • C:\R 폴더에 프로그램을 설치합니다.
    • 윈도운 검색에서 시스템 환경변수 편집을 검색합니다
    • 시스템 속성 판넬에서 맨아래 환경 변수를 클릭합니다.
    • 사용자 변수에서 새로 만들기 클릭하여 RTOOLS43_HOME 환경변수 생성하고 생성값으로 C:\R\rtools43\를 입력합니다.
    • 시스템변수의 path 변수에 편집을 클릭하고 %RTOOLS43_HOME%\usr\bin 추가 합니다.
    • RStudio 설치후 Sys.which("make") 명령어로 RTools 연결되었는지 확인합니다.

    RStudio 프로그램 설치

    • RStudio 공식 웹사이트(https://www.rstudio.com/products/rstudio/download/)에서 RStudio 다운로드 페이지로 이동합니다.
    • 해당 페이지에서 운영 체제에 맞는 버전을 선택하고 다운로드합니다.
    • 다운로드가 완료되면 설치 파일을 실행하여 설치 프로세스를 시작합니다.
    • 설치 마법사의 지시에 따라 설치를 진행합니다. 기본 설정을 유지하거나 사용자 지정 설정을 선택할 수 있습니다.
    • 설치가 완료되면 RStudio가 시스템에 설치됩니다.

    설치 절차

    • https://posit.co/에 접속합니다.
    • RStudio 홈페이지의 화면 왼쪽의 Download RStudio 클릭합니다.
    • 데스크탑 버전 다운로드합니다.
    • 다운로드 완료 후 관리자 모드로 프로그램 설치합니다.
    • 프로그램 설치시 기본 설정 값으로 다음 클릭합니다.
    • R 프로그램의 설치 위치인 C:\R 안에 설치합니다.
    • 모두 체크하고 다음 버튼 클릭합니다.

    miniconda 프로그램 설치

    • 먼저, 인터넷 브라우저를 열고 Miniconda의 공식 웹사이트인 https://docs.conda.io/en/latest/miniconda.html 에 접속합니다.
    • 다운로드 페이지에서 사용하는 운영 체제에 맞는 Miniconda 설치 파일을 선택하여 다운로드합니다. 대부분의 경우 Windows, macOS 및 Linux용 설치 파일이 제공됩니다.
    • 설치 파일을 다운로드한 후에는 해당 파일을 실행하여 설치 프로세스를 시작합니다.
    • 설치 중에는 사용자 지정 옵션을 선택할 수 있습니다. 대부분의 경우 기본값으로 설정해도 괜찮지만, 필요에 따라 경로나 환경 변수 설정 등을 변경할 수 있습니다.
    • 설치가 완료되면 명령 프롬프트(Windows) 또는 터미널(맥 또는 리눅스)을 열고 conda --version 명령을 실행하여 설치가 올바르게 되었는지 확인합니다.
    • 이제 Miniconda가 설치되어 파이썬 환경을 관리할 수 있습니다. 필요한 Python 패키지를 설치하거나 가상 환경을 생성하여 프로젝트를 관리할 수 있습니다.

    python 설치시 miniconda 설치 이유

    • 프로젝트 별로 가상 환경을 만들 수 있습니다.
    • python의 다른 버전을 각각의 프로젝트별로 설치 가능합니다.
    • 프로젝트별 패키지 버전관리가 가능합니다.

    설치 절차

    • Miniconda 홈페이지에서 다운로드합니다.
    • 다운로드 후 인스톨 실행 [Next]클릭 다음하면 [I Agree] 클릭합니다.
    • 인스톨타임 All Users 산택 후 [Next] 클릭합니다.
    • 인스톨 경로 디폴트값으로 선택하고 [Next] 클릭합니다.
    • 인스톨 옵션창에서 Path 설정 체크박스 모두 체크 후 [Install] 클릭합니다.
    • 설치 완료되면 Win+R 를 칩니다.
    • 실행창에 cmd 입력한 후 Enter 를 칩니다.
    • cmd창 열리면 'python' 입력한 후 Enter 후 파이선 작동하는지 확인합니다.

    Miniconda 사용하는 장점

    • 간편한 설치: Miniconda는 간단한 설치 프로세스를 통해 빠르게 설치할 수 있습니다. 불필요한 패키지나 도구를 제거하고 싶은 경우 Miniconda를 선택할 수 있습니다.
    • 가상 환경 관리: Miniconda를 사용하면 가상 환경을 생성하고 관리할 수 있습니다. 각각의 프로젝트에 맞는 독립적인 환경을 만들어 패키지 의존성을 격리할 수 있습니다.
    • 패키지 관리: Conda 패키지 관리자를 사용하여 다양한 Python 패키지를 설치, 업데이트, 제거할 수 있습니다. 이를 통해 패키지 간의 충돌을 방지하고 프로젝트에 필요한 패키지를 쉽게 관리할 수 있습니다.
    • 다양한 운영 체제 지원: Miniconda는 Windows, macOS 및 Linux와 같은 다양한 운영 체제에서 사용할 수 있습니다. 따라서 여러 환경에서 일관된 개발 및 배포 환경을 유지할 수 있습니다.
    • 설정의 최소화: Miniconda는 최소한의 구성만 제공하여 시작할 수 있으며 필요에 따라 필요한 패키지를 추가로 설치할 수 있습니다. 이를 통해 불필요한 설정을 최소화하고 개발 프로세스를 간소화할 수 있습니다. 이러한 장점들은 Miniconda를 사용하여 Python 개발 및 데이터 과학 프로젝트를 보다 효율적으로 관리하고 실행할 수 있게 해줍니다.

    직접 miniconda에서 작업(CMD에서 작업)

    • Win+R 실행창에 cmd 입력한 후 Enter

    • miniconda 실행해서 가상공간(홈페이지 작업 가상공간 ) 생성 

      C:\users\psn\conda create -n myfirstenv python=3.10

    • 만들어진 가상공간으로 입장 
      C:\users\psn\conda activate myfirstenv
    • 들어간 가상공간에서 설치된 프로그램 패키지 확인 
      (myfirstenv) C:\users\psn\conda list
    • 들어간 가상공간에서 프로그램 패키지 설치 
      (myfirstenv) C:\users\psn\pip install mkdocs
    • 만들어진 가상공간으로 나오기 
      (myfirstenv) C:\users\psnam\conda deactivate

    • 만들어진 가상공간 삭제 

      C:\users\psnam\conda env remove -n myfirstenv

    • 만든 가상공간 리스트 확인 

      C:\users\psnam\conda env list

    R에서 Miniconda 사용

    • reticulate 설치: R에서 Python을 사용하기 위해 reticulate 패키지를 설치합니다. 이를 통해 R과 Python을 연결하고 Python 패키지 및 환경을 사용할 수 있습니다.
    install.packages("reticulate")

    Python 인터프리터 지정: use_python() 함수를 사용하여 R에서 사용할 Python 인터프리터의 경로를 지정합니다. 이때 Miniconda가 설치된 경로를 지정해야 합니다.

    library(reticulate)
use_python("/path/to/miniconda/bin/python")

    • 이때 /path/to/miniconda/bin/python은 Miniconda가 설치된 디렉토리 내의 Python 인터프리터의 경로를 나타냅니다. 보통은 Windows에서는 C:\Users\username\Miniconda3\python.exe와 같은 경로가 될 수 있습니다.

    • 가상 환경 생성: 필요한 경우 virtualenv_create() 함수를 사용하여 가상 환경을 생성할 수 있습니다.

    virtualenv_create(envname = "myenv")
    - 이때 myenv는 가상 환경의 이름을 나타내며 원하는 이름으로 변경할 수 있습니다. 가상 환경 활성화: 생성한 가상 환경을 활성화하여 사용합니다.

    virtualenv_install("myenv", packages = c("numpy", "pandas"))
    - 이렇게 하면 myenv 가상 환경에 numpy와 pandas 패키지가 설치됩니다.

    • Python 코드 실행: py_run_file() 함수를 사용하여 Python 스크립트를 실행할 수 있습니다.
    py_run_file("script.py")

    • 이때 script.py는 실행하려는 Python 스크립트 파일의 경로를 나타냅니다.

    • 이러한 단계를 따라 R에서 Miniconda를 사용하여 Python 환경을 설정하고 사용할 수 있습니다.

    • Miniconda 환경 설정: R에서 사용할 Miniconda 환경을 설정합니다. 이를 위해 use_condaenv() 함수를 사용합니다. 이 함수는 Miniconda 환경을 활성화하여 해당 환경에 설치된 Python 패키지 및 환경 변수에 액세스할 수 있도록 합니다.

    library(reticulate)
use_condaenv("name_of_conda_environment")
    - 여기서 name_of_conda_environment은 사용할 Miniconda 가상환경의 이름을 나타냅니다. Miniconda를 설치하고 가상 환경을 만들었다면 해당 가상 환경의 이름을 여기에 제공하면 됩니다.

    • Python 패키지 설치: 필요한 Python 패키지를 설치합니다. 이때 reticulate 패키지의 py_install() 함수를 사용합니다.

    py_install("numpy")
py_install("pandas")
    - 위와 같이 필요한 패키지를 설치할 수 있습니다.

    • Python 코드 실행: 필요한 경우 Python 코드를 실행할 수 있습니다. py_run_file() 함수를 사용하여 Python 스크립트를 실행합니다.

    py_run_file("C:/gppj/gppj01/p/gppj01_0403.py")
    - 이때 script.py는 실행하려는 Python 스크립트 파일의 경로를 나타냅니다.

    • 위의 단계를 따르면 R에서 Miniconda를 사용하여 Python 환경을 설정하고 사용할 수 있습니다. 이를 통해 R과 Python을 효율적으로 결합하여 다양한 분석 및 개발 작업을 수행할 수 있습니다.

    R에서 Miniconda를 사용하는 다른 방법

    R에서 Miniconda를 사용하는 방법은 다양합니다. 그러나 여기에 몇 가지 추천 사항이 있습니다.

    • reticulate 패키지 사용: R에서 파이썬 코드를 실행하는 가장 일반적인 방법은 reticulate 패키지를 사용하는 것입니다. 이 패키지는 R에서 파이썬 코드를 실행하고 파이썬 패키지를 로드할 수 있도록 해줍니다. Miniconda를 설치하고 파이썬 환경을 관리한 다음 reticulate를 사용하여 해당 환경을 호출할 수 있습니다.

    • conda 패키지 관리자 사용: R에서 conda 패키지 관리자를 사용하여 Miniconda 환경을 관리할 수도 있습니다. 이 방법은 conda를 직접 R 코드에서 호출하여 파이썬 패키지를 설치하고 관리하는 것입니다.

    • renv와 함께 사용: renv는 R 프로젝트의 환경을 관리하는 패키지로, 파이썬 프로젝트와 동일한 방식으로 R 환경을 관리할 수 있습니다. Miniconda를 사용하여 Python 환경을 만들고 renv를 사용하여 해당 환경을 관리하는 것이 좋은 방법입니다.

    • 이러한 방법 중에서는 reticulate를 사용하는 것이 가장 간단하고 직관적입니다. 다른 방법들은 보다 고급 사용자를 위한 것이며, 프로젝트의 요구 사항 및 개발자의 선호도에 따라 선택할 수 있습니다.
